# 🕒 NTP Time Service — Master Plan

## Goal
Build a robust time service in VPre CountdownCtrl that:
- **Fetches real time** from a configurable HTTP time server (default: worldtimeapi Hong Kong)
- **Computes an offset** between server time and local clock
- **Fails over gracefully** to local clock when NTP is unreachable (preserving last known offset)
- **Shows clock status** visibly in the status bar
- **Allows NTP server configuration** from the Settings page

## Architecture

### NTPManager (JS Object)
```
NTPManager
├── config (from appConfig/localStorage):
│   ├── ntpServerUrl       // default: worldtimeapi HKG
│   ├── ntpAutoSyncInterval // seconds (0 = off)
│   ├── ntpLastSyncTime     // ISO string
│   └── ntpLastOffset       // milliseconds
├── status: 'connected' | 'fallback' | 'local' | 'syncing' | 'error'
├── sync(): Promise<void>   // fetch → parse → compute offset
├── getCalibratedDate(): Date // existing fn, now uses timeOffset
└── updateStatusUI(): void // update DOM badges
```

### Data Flow
```
syncTime(url)
  ├─ fetch(url, timeout=5s)
  │   ├─ Success → parse datetime → offset = serverEpoch - Date.now()
  │   │             → save timeOffset, ntpLastOffset, ntpLastSyncTime
  │   │             → updateStatusUI('connected')
  │   └─ Failure → keep last offset → updateStatusUI('fallback'|'local')
  │
  getCalibratedDate()
    └─ base = Date.now() + timeOffset → timezone conversion
```

## Non-Goals
- Raw NTP (UDP 123) — not possible from browser
- Server-side proxy — out of scope
- Multiple simultaneous NTP sources — single server + fallback is sufficient
