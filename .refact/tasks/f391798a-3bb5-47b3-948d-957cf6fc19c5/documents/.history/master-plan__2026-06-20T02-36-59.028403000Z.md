# 🐍 Python Backend + NTP Time Service — Master Plan

## Goal
Convert VPre CountdownCtrl from pure-SPA (localStorage + HTTP API) to a **Python-backed web application** with real NTP time sync, persistent SQLite database, and one-command startup (`python server.py`).

## Architecture

```
┌─────────────────────────────────────┐
│   Python FastAPI Server (server.py) │
│                                     │
│  ┌───────────┐  ┌────────────────┐  │
│  │ ntplib    │  │ SQLite         │  │
│  │ time.hko  │  │ schedules      │  │
│  │ .hk (NTP) │  │ presets        │  │
│  │           │  │ config         │  │
│  └───────────┘  └────────────────┘  │
│         │                │          │
│    /api/ntp/*     /api/schedule/*   │
│                   /api/preset/*     │
│                   /api/config       │
│                   /api/backup       │
│                                     │
│  Serves: templates/index.html       │
└──────────────┬──────────────────────┘
               │ fetch(JSON)
┌──────────────┴──────────────────────┐
│  Browser (Frontend SPA)             │
│                                     │
│  - CRUD → API calls (fallback to    │
│    localStorage if server unreach.) │
│  - NTP   → /api/ntp/sync            │
│  - Engine (CUE, matrix, timeline)   │
│    unchanged                        │
└─────────────────────────────────────┘
```

## Files
| File | Purpose |
|------|---------|
| `server.py` | FastAPI server (single file): routes, DB, NTP, static serving |
| `requirements.txt` | Python dependencies |
| `templates/index.html` | Copy of SPA, modified to call backend API |
| `vcc_pre.db` | SQLite database (auto-created) |

## Cards

| Card | Title | Depends | Verification |
|------|-------|---------|-------------|
| BE-1 | Backend Core + NTP Service | — | `python server.py` starts, `curl /api/ntp/status` returns valid time |
| BE-2 | Schedule & Preset CRUD API | BE-1 | `curl /api/schedule` returns data from SQLite |
| BE-3 | Config & Backup API | BE-2 | Config save/load persists across restarts |
| FE-1 | Frontend API Integration | BE-3 | All CRUD operations go through backend API |

| FE-3 | Docs + Polish | FE-2 | README shows one-line setup |

## Non-Goals
- User authentication/login
- Docker deployment (optional future)
- Multi-master replication
