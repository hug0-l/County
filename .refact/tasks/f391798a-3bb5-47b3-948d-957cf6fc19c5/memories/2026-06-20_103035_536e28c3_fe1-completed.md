# FE-1 Completed

FE-1 Frontend API Integration completed successfully.

## What was implemented

1. **API Client wrapper** (`const API = {...}`) — inserted after NTPManager, before GUI calendar section
   - `_fetch()`: generic fetch with 3s `AbortSignal.timeout(3000)`, returns null on failure
   - Schedule CRUD: `loadSchedules()`, `saveSchedule()`, `updateSchedule()`, `deleteSchedule()`
   - Preset CRUD: `loadPresets()`, `savePreset()`, `updatePreset()`, `deletePreset()`
   - Config: `loadConfig()`, `saveConfig()`
   - NTP: `ntpStatus()`, `ntpSync()`
   - Backup: `downloadBackup()`, `restoreBackup()`
   - Legacy: `importLegacy()`

2. **Normalizer helpers**: `normalizeScheduleFromApi()` (snake_case→camelCase), `normalizePresetFromApi()`, `formatForServer()` (camelCase for POST)

3. **Page load data fetch**: `fetchApiData()` called after NTP sync in `syncWithNetworkTime()` callback — fetches schedules, presets, config from API, merges into memory/localStorage

4. **CRUD sync to server (fire-and-forget)**:
   - `saveToLocalStorage()` → calls `syncAllToServer()` (POST all schedules + presets)
   - `savePresetAction()` → calls `API.savePreset()` after localStorage save
   - `saveConfig()` → calls `API.saveConfig()` after localStorage save
   - `applyConfigJson()` → calls `API.saveConfig()` after localStorage save
   - `deleteProgram()` → calls `API.deleteSchedule()` before localStorage save
   - `clearAllPrograms()` → calls `API.deleteSchedule()` for each program

5. **API-first backup/restore**:
   - `saveDataToFile()` → tries `API.downloadBackup()` first, falls back to localStorage export
   - `handleFileOpen()` → tries `API.restoreBackup()` first, falls back to localStorage import

## Key design decisions
- All API calls are non-blocking fire-and-forget (`.then()`, no `await`)
- localStorage is always the primary store; server is secondary
- 3-second timeout on API calls; null returned on any failure
