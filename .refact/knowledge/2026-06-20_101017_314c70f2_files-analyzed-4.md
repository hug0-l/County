# Solution
我們轉向 **Python Backend + 開罐即用** 架構，讓整套排控系統成為一個本地可執行的伺服器，前端（原 VPre CountdownCtrl）變成它的用戶端。以下說明「如何使用」這套新架構，以及我們要實作的規劃。

---

## 🧱 新架構：Python Server + 前端 SPA

```
┌──────────────────────────────────────┐
│         Python 後端 (FastAPI)         │
│                                      │
│  • REST API (JSON)                   │
│  • ntplib → 直接 UDP NTP 校時        │
│    (time.hko.hk / time.stdtime.gov.tw)│
│  • SQLite 資料庫 (schedule, presets,  │
│    config, ntp_log)                  │
│  • Static files → serve 前端 HTML    │
│                                      │
└───────┬──────────────────────────────┘
        │
┌───────┴──────────────────────────────┐
│        瀏覽器 (前端 SPA)              │
│                                      │
│  原本的 VPre_CountdownCtrl_PROD_v0.5 │
│  改為：                              │
│  - localStorage → API calls (fetch)  │
│  - NTP 同步 → /api/ntp/sync          │
│  - 排程 CRUD → /api/schedule         │
│  - 設定載入 → /api/config            │
│  - 備份/還原 → /api/backup           │
└──────────────────────────────────────┘
```

### 如何使用（操作員視角）

1. **安裝 Python** 3.10+，從 GitHub 拉取倉庫。
2. **執行一條指令**：
   ```bash
   python server.py
   ```
   伺服器自動啟動，瀏覽器打開 `http://localhost:8000` 就是你的主控台。
3. **NTP 自動校時** — server 每 10 分鐘對 time.hko.hk 做真實 NTP 同步，精度 ±5ms。
4. **多個操作員** 同一網路內用不同電腦開相同 URL，即時共享同一個排程資料庫。
5. **資料安心** — 所有排程、Preset、Config 存在 SQLite，不怕瀏覽器快取清除，也能定時自動備份。

## 🔁 前端修改範圍

原本 localStorage 的操作全部改為 `fetch` API calls：

| 原本 | 改為 |
|------|------|
| `localStorage.setItem('vcc_pre_master_db_v8', ...)` | `fetch('/api/schedule', {method:'PUT', body:JSON.stringify(db)})` |
| `localStorage.getItem('vcc_pre_config_v8')` | `fetch('/api/config').then(r=>r.json())` |
| `NTPManager.sync(worldtimeapi)` | `fetch('/api/ntp/sync')` (server 用 ntplib 算好 offset 回傳) |
| 備份匯出 | `fetch('/api/backup/download')` |
| 備份匯入 | `fetch('/api/backup/restore', {method:'POST', body: file})` |

> **核心引擎不變** — CUE 觸發、時間軸、矩陣計算仍全在瀏覽器端，不受影響。
