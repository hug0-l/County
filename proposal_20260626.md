## 二、Item-based Cue 系統（節目段落）

### 2.1 問題

目前 Cue 點僅存在於「節目層級」（開始前、開始後、結束前、結束時）。實際播出一檔 30 分鐘的新聞可能包含：

- Headlines（頭條）
- Weather（天氣）
- Sports（體育）
- Ad Break（廣告）
- Interview（專訪）

**每個段落都需要自己的 Cue 點**（例如「天氣段落前 15 秒提示」），但現行系統無法表達，操作員只能把段落拆成多筆排程，失去節目整體性。

### 2.2 資料模型

在 Preset 中新增 optional `items` 陣列：

```javascript
{
  name: "新聞節目",
  nodes: [
    // 節目層級 Cue（相對於節目開始/結束）
    { offset: -300, name: "⏳ 開始前5分", soundId: "bell" },
    { offset: 0,    name: "▶ 開播",      soundId: "alert" },
    { offset: "e-30", name: "🔔 結束前30秒", soundId: "double_beep" },
    { offset: "e0",   name: "■ 結束",    soundId: "bell" }
  ],
  items: [
    {
      name: "📰 Headlines",
      duration: 300,  // 5 分鐘（秒為單位）
      nodes: [
        { offset: -15, name: "🔔 Weather 前15秒", soundId: "triple_beep" }
      ]
    },
    { name: "🌤️ Weather", duration: 300, nodes: [] },
    { name: "🏃 Sports",   duration: 300, nodes: [] }
  ]
}
```

### 2.3 引擎計算邏輯

- Item 開始時間 = 節目開始 + 前面所有 Item duration 總和
- Item 結束時間 = Item 開始 + Item.duration
- Item Cue 時間 = Item 開始 + node.offset
- 完全向後相容（`items` 為 optional，舊 Preset 不影響）

### 2.4 Preset 編輯器 — Item 模式

```
┌─ Preset: 新聞節目 ─────────────────────────────┐
│                                                  │
│ 📋 節目層級 Cue                                  │
│ ┌────────────────────────────────────────────┐  │
│ │ 開始前 -300s  ⏳ 開始前5分      bell  🔊 ✕ │  │
│ │ 開始後 0s     ▶ 開播           alert 🔊 ✕ │  │
│ │ 結束前 e-30   🔔 結束前30秒    chime  🔊 ✕ │  │
│ └────────────────────────────────────────────┘  │
│                                                  │
│ 📋 段落 Items                                    │
│ ┌────────────────────────────────────────────┐  │
│ │ ▸ 📰 Headlines      05:00    ✕   ↕         │  │
│ │   ├ 開始前 -15s  🔔 前15秒  triple_beep    │  │
│ │   └ 開始後 0s    ▶ Weather  double_beep    │  │
│ │ ▸ 🌤️ Weather        05:00    ✕   ↕         │  │
│ │ ▸ 🏃 Sports         05:00    ✕   ↕         │  │
│ │ [+ 新增 Item]                               │  │
│ └────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────┘
```

- Item 列可收合（▸/▾），預設展開
- Item 子列為 item-level Cue（格式與節目層級相同）
- duration 以 MM:SS 輸入，內部存秒
- 拖曳 ↕ 調整 Item 順序

## 三、操作員為中心 — Live 儀表板

### 3.1 核心原則

現場操作員只需要回答三個問題：

1. **現在**什麼狀況？
2. **下一個**是什麼時候？
3. 我需不需要**改什麼**或**通知誰**？

其餘資訊應收合為次要層級。

### 3.2 三層儀表板

```
┌─ L1: 🟢 狀態列 ────────────────────────────────┐
│  NORMAL · 07:00:00:00 · 📡 ON AIR: 新聞報導    │
├─ L2: ON AIR Card ──────────────────────────────┤
│  剩餘 22:34        07:00 → 07:30                │
│  [ -30s ] [ +30s ] [ +1m ] [ +5m ]  [🔊 通知]  │
│  開播: 07:00 📝         結束: 07:30 📝           │
├─ L3: Next Cue ─────────────────────────────────┤
│  ➔ NEXT: 🌤️ 天氣預報  07:30  (-08:00)          │
│  ⏱ CUE: Weather 前15秒                         │
│  ██████████████████░░░░░░░  00:45               │
├─ [▾ 完整時序矩陣] ─────────────────────────────┤
│  (預設收合，點擊展開)                            │
└──────────────────────────────────────────────────┘
```

**L1 — 狀態列**（固定，永遠可見）：
- 4px 頂部顏色條（🟢正常 / 🟡Cue 將至 / 🔴Cue 觸發 / ⚪引擎關閉）
- 主時鐘 + 當前播出節目

**L2 — ON AIR Card**（開播後 30 秒自動收合為標題列）：
- 剩餘時間、一鍵偏移按鈕、一鍵通知、inline 時間編輯

**L3 — Cue 進度條 + Next**：
- 進度條取代純數字倒數（CSS transition 平滑動畫）
- < 30s 轉紅閃爍

## 四、一鍵時間偏移（最高優先）

### 4.1 現狀

Quick Edit 需要點三下：按鈕 → 修改時碼 → 套用確認。現場最常見的操作「延後 1 分鐘」太慢。

### 4.2 建議

ON AIR Card 直接放置偏移按鈕：

```
[ -30s ] [ +30s ] [ +1m ] [ +5m ]
```

- 點擊後該節目 + 所有後續節目一起平移
- 自動衝突檢測
- 5 秒內可 Undo
- 自動產生日誌 + 可選 Clipper 通知

### 4.3 實作

```javascript
function shiftProgram(progId, offsetSec) {
  const prog = masterScheduleDB.find(p => p.id === progId);
  const laterProgs = getExpandedRundownForDate(prog.broadcastDate)
    .filter(p => timecodeToTotalFrames(p.startTime) > timecodeToTotalFrames(prog.startTime));
  window._lastShiftUndo = { progId, offsetSec, targets: laterProgs.map(p => ({ id: p.id, oldStart: p.startTime })) };
  [prog, ...laterProgs].forEach(p => {
    p.startTime = totalFramesToTimecode(timecodeToTotalFrames(p.startTime) + offsetSec * frameRate);
  });
  saveToLocalStorage();
  scheduleMatrixRecalc();
}
```

## 五、一鍵 Clipper 通知

### 5.1 建議

每個 ON AIR 節目旁的 `[🔊 通知]` 按鈕，點擊後彈出 4 個快速模板：

1. 📡 {name} 已開播，預計 {endTime} 結束
2. ⏱ {name} 延後 {n} 分鐘 → 預計 {newEndTime} 結束
3. ⏱ {name} 提前 {n} 分鐘 → 預計 {newEndTime} 結束
4. ✏️ 自訂訊息

點擊後自動發送至 Clipper 當前房間，離線時退回剪貼簿。

### 6.1 建議

ENGINE LIVE 時：

| 頁面 | 狀態 |
|------|------|
| 側欄 | 🟢 ENGINE ON-AIR |

## 七、Cue 進度條

### 7.1 建議

以進度條取代當前純數字倒數：

```
⏱ CUE: Weather 前15秒
██████████████████░░░░░░░  00:45
```

- CSS transition 1s linear 平滑動畫
- < 30s 條轉紅 + 閃爍
- 點擊進度條 → 精確秒數 tooltip

```css
.cue-progress-fill { background: var(--warning); transition: width 1s linear; }
.cue-progress-fill.urgent { background: var(--danger); animation: pulse 0.5s infinite; }
```
